# おすすめ自動取得の件数増加機能

## Goal Description
現在、自動取得の件数が比較的少ないため、楽天およびYahooショッピングからの1回あたりの取得件数を大幅に増やす改修を行います。

## Proposed Changes

### Discovery Feature
各外部API検索サービスの設定を変更・拡張し、より多くのアイテムを取得できるようにします。

#### [MODIFY] RakutenItemSearchService.java
楽天APIは1リクエストあたりの最大取得件数（`hits`）が30件に制限されています。そのため、ページネーション（`page`パラメータ）を利用して、各ジャンルにつき1ページ目〜3ページ目（合計90件）を取得するループ処理を追加します。
- 予想取得件数: 7ジャンル × 30件 × 3ページ = 最大630件

#### [MODIFY] YahooItemSearchService.java
YahooショッピングAPIは1リクエストあたりの最大取得件数（`results`）を100件まで設定できます。現在の30件設定を100件に引き上げることで、ループを増やさずに1回で大量のデータを取得します。
- 予想取得件数: 7カテゴリ × 100件 = 最大700件

## Verification Plan

### Manual Verification
1. 修正後、ローカルでアプリケーション(`bootRun`)を再起動する。
2. 画面から「🤖 おすすめ自動取得」ボタンを再度クリックする。
3. 取得完了時のアラートでの取得件数、またはデータベース(`discovery_items`テーブル)の件数が前回よりも大幅に増加していることを確認する。
